command deck

Everything it takes requests for

Used from any voice channel in the server. Anyone who can run slash commands can drive it — there's no login, no permission tiers, no gatekeeping.

/tplay <song>
Starts music, replacing the current song but keeping the queue.
/tqueue <song>
Adds a song after the one that's playing.
/tqueue
Shows what's playing and what's up next.
/tremove <position>
Removes one queued song by its number.
/tskip
Moves on to the next queued song.
/tloop
Loops the current song.
/tstoploop
Turns looping off so the queue keeps moving.
/tpause · /tresume
Pauses playback, or picks it back up.
/tnowplaying
Shows the current song, progress, and who requested it.
/treplay
Replays the previous song.
/trandom
Plays a random song from this session's history.
/thistory
Lists every song played so far this session.
/tartist <name>
Keeps queueing more by that artist whenever the queue runs dry.
/tstopartist
Turns artist mode back off.
/trelated
Finds and queues a song related to what's playing.
/tclear
Clears the queue without stopping the current song.
/tstop
Stops playback and clears everything.

Get it running

Five steps from a clone to a bot sitting in your voice channel.

1
git clone https://github.com/josephevenson08/TunezBot.git
2
cd TunezBot && npm install
3
cp .env.example .env
4
npm run deploy
5
npm start
Step 3 needs real values before step 4 — DISCORD_TOKEN, CLIENT_ID, and GUILD_IDS from the Discord Developer Portal. The README in the repo walks through getting each one.
